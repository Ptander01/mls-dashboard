# MLS Dashboard Documentation

This directory contains all project documentation, specifications, research, and content drafts for the MLS Analytics Dashboard. 

The structure is organized to keep project management, technical specifications, and content creation separated and easy to navigate.

## Directory Structure

- **`/blog`** — LinkedIn post drafts, article content, and feature release announcements.
- **`/design-system`** — Technical specifications and HTML wireframes for the dashboard's neumorphic UI and chart architecture.
- **`/reports`** — Daily progress reports, backlog audits, project status syntheses, and brainstorming feedback.
- **`/research`** — Data source notes, StatsBomb inventory, and reference material for specific features (e.g., travel map, ridgelines).
- **`/sprints`** — Sprint briefs (`/briefs`) and completed handoff documents (`/handoffs`) used to guide development sessions.
- **`/verification`** — QA and verification plans for major feature integrations.
- **`/assets`** — High-resolution hero shots, screenshots, and architecture diagrams used in documentation and blog posts.

## Key Documents

- **`HANDOFF.md`** — The primary top-level project handoff document. Start here if you are new to the repo.
- **`../BACKLOG.md`** — The master prioritized backlog and work plan (located in the repository root).
- **`../ARCHITECTURE.md`** — The technical architecture and self-hosting guide (located in the repository root).
