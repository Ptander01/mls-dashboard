TripsLayer — Animated Travel Arcs
This is the showstopper. Travel routes animate as flowing particles synchronized to the matchweek scrubber:

import { TripsLayer } from '@deck.gl/geo-layers'

// Transform match data into trips format
// Each trip = one away game journey
const tripData = useMemo(() => {
  return matches
    .filter(m => m.week <= currentWeek)
    .map(match => {
      const awayTeam = teams.find(t => t.id === match.awayTeam)
      const homeTeam = teams.find(t => t.id === match.homeTeam)
      if (!awayTeam || !homeTeam) return null
     
      // Arc waypoints with timestamps
      // Midpoint elevated to create arc height
      const midLng = (awayTeam.lng + homeTeam.lng) / 2
      const midLat = (awayTeam.lat + homeTeam.lat) / 2
     
      return {
        vendor: match.awayTeam,
        color: hexToRgb(awayTeam.color),
        timestamps: [match.week * 100, match.week * 100 + 50, match.week * 100 + 100],
        path: [
          [awayTeam.lng, awayTeam.lat, 0],
          [midLng, midLat, 150000],  // arc peak height in meters
          [homeTeam.lng, homeTeam.lat, 0]
        ]
      }
    })
    .filter(Boolean)
}, [matches, currentWeek])

// Animate current time
const [currentTime, setCurrentTime] = useState(0)
useEffect(() => {
  if (!isPlaying) return
  const interval = setInterval(() => {
    setCurrentTime(t => {
      if (t >= currentWeek * 100 + 100) {
        clearInterval(interval)
        return t
      }
      return t + 2
    })
  }, 16)
  return () => clearInterval(interval)
}, [isPlaying, currentWeek])

The Core Concept
Junyan Zhang’s layered brick aesthetic achieved through ColumnLayer in OrthographicView. Each team is one square column. Height = resilience score. Color = tier. The orthographic camera keeps everything perfectly square and page-aligned while WebGL lighting makes each column face respond to the directional light.

Grid Layout
Compute a fixed grid layout — 6 columns × 5 rows for 30 teams, sorted by resilience score descending left-to-right, top-to-bottom:

function computeGridLayout(teams: TeamResilienceMetrics[]) {
  const sorted = [...teams].sort((a, b) => b.resilienceScore - a.resilienceScore)
  const cols = 6
  const cellSize = 120   // px per cell
  const gap = 8          // gap between cells
 
  return sorted.map((team, i) => ({
    ...team,
    gridX: (i % cols) * (cellSize + gap) + cellSize / 2,
    gridY: Math.floor(i / cols) * (cellSize + gap) + cellSize / 2,
  }))
}


Unlike squarified treemap, this gives every team equal tile area — the data encoding is entirely in height and color, not area. This solves the “small tiles are illegible” problem from the current implementation.

The Core Concept
Same ColumnLayer + OrthographicView pattern as the treemap. Each cell in the correlation grid is one square column. Height = |r|. Color = blue (positive) or red (negative). The animation between Raw Counts and Per 90 modes is the centrepiece — columns rise and fall to new heights with a 600ms transition.
Data Transform

function correlationToColumnData(matrix: number[][], statNames: string[]) {
  const columns: ColumnDatum[] = []
  const n = statNames.length
  const cellSize = 60   // px per cell in orthographic space
  const gap = 4
 
  matrix.forEach((row, i) => {
    row.forEach((r, j) => {
      columns.push({
        id: `${i}-${j}`,
        x: j * (cellSize + gap),
        y: i * (cellSize + gap),
        r: r,
        absR: Math.abs(r),
        isPositive: r >= 0,
        isDiagonal: i === j,
        rowLabel: statNames[i],
        colLabel: statNames[j],
      })
    })
  })
 
  return columns
}


ColumnLayer Configuration

const matrixLayer = new ColumnLayer({
  id: 'correlation-matrix',
  data: columnData,
 
  diskResolution: 4,    // square prisms
  radius: 26,           // slightly smaller than half cell
  extruded: true,
 
  getPosition: d => [d.x, d.y],

  getElevation: d => d.isDiagonal ? 2 : d.absR * 80,
 
  getFillColor: d => {
    if (d.isDiagonal) return [120, 120, 140, 180]  // neutral gray
    if (d.isPositive) {
      // Blue scale — deeper blue = stronger positive
      const intensity = Math.floor(d.absR * 180)
      return [40 - intensity * 0.1, 80, 180 + intensity * 0.4, 230]
    } else {
      // Red scale — deeper red = stronger negative
      const intensity = Math.floor(d.absR * 180)
      return [180 + intensity * 0.4, 50, 50, 230]
    }
  },
 
  material: {
    ambient: 0.3,
    diffuse: 0.85,
    shininess: 12,
    specularColor: [200, 210, 255]
  },
 
  pickable: true,
  onClick: ({ object }) => {
    if (!object.isDiagonal) {
      // Set scatter plot axes to this stat pair
      setScatterAxes(object.rowLabel, object.colLabel)
      scrollToScatter()
    }

Critical Notes for Manus
Test on mobile after each session. deck.gl WebGL is GPU-dependent. Some mobile browsers throttle WebGL contexts aggressively. If performance is poor on mobile, add useDevicePixels: false to the DeckGL component to render at 1x instead of device pixel ratio — this halves the GPU workload.
The _shadow: true flag on DirectionalLight is expensive. If frame rate drops below 30fps, remove it first. The lighting quality degrades but performance recovers immediately.
CARTO tile rate limits. The free CARTO basemap tiles have rate limits for production use. For a portfolio project the free tier is fine, but if the map tiles fail to load add a fallback: onError={() => setMapStyle(FALLBACK_STYLE)}.
Terrain tiles need CORS headers. The AWS elevation tile endpoint should work but if you see CORS errors in the console, replace with Mapbox terrain tiles — those have reliable CORS configuration. You’ll need a free Mapbox token but the terrain tile endpoint is included in the free tier.​​​​​​​​​​​​​​​​
